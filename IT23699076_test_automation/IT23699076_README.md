# IT23699076 – IT3040 Assignment 1
## Automated Testing for Singlish-to-Sinhala Transliteration System

This project implements a robust automated testing framework using Python and Playwright to validate the Singlish-to-Sinhala transliteration tool available at [Pixelssuite Chat Translator](https://www.pixelssuite.com/chat-translator).

---

## 📂 Project Structure

The repository is organized as follows:

- **`IT23699076_test_automation.py`**: The core automation engine. It uses Playwright to interact with the web UI and OpenPyXL to read/write test data.
- **`IT23699076_Assignment 1 - Test cases.xlsx`**: The test suite containing 50+ test cases across various categories (Question forms, Command forms, Requests, etc.).
- **`IT23699076_Requirements.txt`**: Lists the necessary Python dependencies for the project.
- **`IT23699076_README.md`**: Project documentation and execution guide.

---

## 🛠️ Technologies & Features

### Core Stack
- **Python 3.10+**: Base programming language.
- **Playwright**: Modern browser automation for reliable UI testing.
- **OpenPyXL**: Advanced Excel manipulation, including support for merged cells and dynamic column detection.

### Key Features
- **Dynamic Header Detection**: Automatically identifies "Input", "Expected Output", and "Status" columns regardless of their position.
- **Merged Cell Support**: Handles complex Excel templates where multiple rows might be merged.
- **Intelligent Wait Logic**: Implements retry mechanisms and change detection to ensure the transliteration is complete before capturing results.
- **Flexible CLI**: Supports various command-line arguments to customize execution (headless mode, slow-mo, custom URLs, etc.).

## Getting Started

### 1. Prerequisites
Ensure you have Python installed. It is recommended to use a virtual environment.

### 2. Installation
Install the required dependencies and Playwright browsers:
```bash
pip install -r IT23699076_Requirements.txt
playwright install chromium
```

### 3. Running the Automation
Execute the script using the following command:
```bash
python IT23699076_test_automation.py --excel "IT23699076_Assignment 1 - Test cases.xlsx" --wait-ms 5000 --type-delay-ms 50
```

#### Common Arguments:
- `--excel`: Path to your test case file.
- `--headless`: Run without showing the browser window.
- `--slow-mo-ms`: Delay between actions (useful for debugging).
- `--keep-open`: Keeps the browser open after completion.

## Test Scenarios Covered

The test suite focuses on **Negative and Edge Case Testing** to evaluate the robustness of the transliteration engine:
- **Linguistic Variations**: Singlish vs. formal Romanized Sinhala.
- **Mixed Content**: Sentences containing English technical terms, digital slang, and abbreviations.
- **Symbols & Emojis**: Verification of how the system handles non-alphanumeric characters.
- **Numeric Data**: Handling of currency (USD/LKR), time formats (AM/PM), and measurements (kg/km).
- **Online Identifiers**: Validation of emails and social media handles (`@username`).
